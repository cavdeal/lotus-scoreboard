A — What is this scoreboard?

This scoreboard tracks all the key numbers across Lotus Development. It has 6 sections (tabs). Each number has a target, an actual, a status, an owner, and a consequence rule. When a number is green, the system is working. When a number is red, the owner must explain and show a plan — the same day.

This scoreboard works the same way at every Lotus Development site. If a new site opens tomorrow, the same tabs, the same rules, and the same cadence apply — no setup needed.

F — What happens when a number is red?

  1. The KPI owner sees the red number (or is told by the system or Team Leader).
  2. The owner tells their Team Leader the same day.
  3. The Team Leader writes a plan: What will you do? By when? Who will check?
  4. The Team Leader presents the plan at the next L10 meeting.
  5. If there is no plan after 24 hours, the GM follows up with the Team Leader directly.
  6. If the same number is red for 2 straight weeks with no improvement, the GM escalates to the MD.
The rule is simple: red with a plan = manageable. Red without a plan = escalate. The system tracks this — nothing falls through the cracks.
